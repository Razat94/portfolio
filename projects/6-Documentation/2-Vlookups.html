A) Getting Started

VLOOKUP is a premade Excel & Google Sheets lookup function that can find information in a row.
I.E. Given any row, I can find information from other columns in that row.

FYI: XLOOKUPs are the new & improved VLOOKUP.
XLOOKUP addresses and improves on the limitations of Vlookup, thereby making it easier to use.

For a vlookup to work, the data set must be structured such that each record/entry is in a separate row.

E.g. Given someones ID Number, find their domain handle.
# First Last Handle
1 Mark Otto @mdo
2 Jacob Thornton @fat
3 Larry Bird @twitter

In an inventory/product list, VLOOKUP can help you find an item's price or details using a part number. It can also be handy for getting a specific employees info (Name, Email, Department, etc.) from a badge number.

Danger #1: For Vlookups to work, the lookup value MUST be in the 1st column! E.G, The badge number must be in the first column otherwise the function will not work.

The Xlookup function overcomes this issue.
Danger #2: Columns must be adjacent i.e. there can be no empty columns in your data set.

The Xlookup function overcomes this issue.

Now, let’s go over the VLOOKUP Function Syntax.

This function takes in four arguments: lookup value, table array, column index number, and range lookup.

  • Lookup Value: The value you want to search for. Remember that this identifier must be in the first column.
  • Table Array: The dataset(range) you’re working with and will use for the search.
  • Column Index Number: Based/Relative on the table array, this number represents the column from which you want to retrieve the value.
  • Range Lookup: Indicates whether you're looking for an exact(FALSE) or approximate(TRUE) match.

The following examples will help examine this function better. The key is not to overcomplicate things since all this function does is return a value from another column.


B) Example #1:

Let's look at the "Example 1" Worksheet

. You can open the file by:
  1. Going to the file directory.
  2. Open the file named "Vlookup Examples"
  3. Open up "Example 1".

Suppose we have a table of some produce as seen below:

Each produce has a price per pound and availability.
Produce Available? Price
Apple Yes $1.50
Banana No $2.03
Lemon No $3.10
Orange Yes $1.01

Since this example is simplified, we can find the price of the produce just by looking at the table. However, suppose we are SuperKing Markets & we sell 100s of produce. It would be very daunting to scan row by row! So instead we can create what's called a "Lookup Table" to find out answer. Below is an example:

Lookup Table
Produce Insert Fruit.
  • Apple
  • Orange
  • Banana
Price? -
Lemon -
Remember: A VLOOKUP simply looks up related values in a row.

In this example, cell B1 will hold our lookup value, and cells B2 and B3 will contain VLookups that output the price and stock availability. As we change the produce listed in B1, the values in B2 and B3 will update automatically.

In short, if given a fruit, we can use VLOOKUP to find both its price and stock status.
So if we want to find the price of Oranges, we’ll use Orange as our lookup value to find its price and stock availability.

It's important not to overcomplicate this function. Always focus on what the output should be.
Remember: If you know the price of ginger, you don't need a VLOOKUP; you can see the price directly just by looking.

The lookup function is Excels way of telling you the answer; it also updates the result based on what’s in B1.

Now, let’s break down how VLOOKUP works:

  • Lookup Value (B1): This is what we're searching for, like cucumbers.
  • Table Array (A7 to C11): This range includes all our data. While you can include the header row, it's usually excluded.
  • Column Index Number: This identifies which column to pull data from—1 for vegetables, 2 for price, and 3 for stock. For price, we use column 2.
  • Match: This would be an EXACT match, since we're looking up exact words.

In our example, the VLOOKUP in B2 returns the price, and the one in B3 shows availability, both based on exact matches.

Question: If we move the data set to another location, will the function still work?

Yes. Because the function still uses cell references. Whether the data set moves or not, the column index number for price is still 2, and the function will still work despite the data range moving.

C) Example #2:

Let's now open up the Example #2 Spreadsheet.

Let's look at another example using a payroll sheet with names, roles, join dates, and total income. Seen below is an example of the data set and the lookup table

Data Source Table
Name Position? Salary
Raza Trainer $1M
Zehra English Teacher $2M
Sarah Director of Training $4M
Lookup Table
Produce Insert Persons Name
  • Raza
  • Zehra
  • Sarah
Price? -
Lemon -

In the example above, when you type a person’s name in F1 in the lookup table, cells F2 and F3, which contain the VLOOKUP functions, will retrieve their role and total income.

When using Vlookup, make sure to spell the names/lookup value correctly or it won’t find a match. For example, If I typed in "Razaaa", I will get a NA error since there is no person named "Razaaaa"

Avoid adding extra characters too like blank spaces.

NOTE! remember to change below:

For example, if you have two cells with similar names (like "Raza" in both A5 and F5), VLOOKUP will return the value from the second column of A5 by default. This can cause issues if you're trying to look up a different name, as it might not return the expected result.
To fix this, you need to ensure your lookup value is correct and that you're focusing on the right data. If your VLOOKUP isn’t working, like in the case of Brenda's name, you might need to update your table array to include all relevant data.

Question: In the data source table, if I enter the wrong name like "Raza Tahir" or "Razaaa" in cell A2, and type "Raza" correctly in the lookup table F1, will the functions still work?

No. Although now we are changing the record entry value, there are no people whose name is exactly "Raza" so the NA error will be given.

The answers below will show what each VLOOKUP should look like:

= VLOOKUP (F1, A2:C4, 2, FALSE)

= VLOOKUP (F1, A2:C4, 3, FALSE)


The only difference between the vlookups is the output; one outputs price and the other output salary.
Therefore, the difference between each functions syntax is the column index number.

Question: In the data source table, if I entered another name, how will our functions change?

By adding an extra name, our data set becomes bigger. So our data array is what changes.

If we were to reference a table by its name, then our function will stay the same yet the table size will change.
